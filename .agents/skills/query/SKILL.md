# query 技能：本地知識網絡深度檢索

## 1. 核心目標
拒絕大模型的幻覺與空泛回答。將用戶的提問精準轉化為對本地 Wiki 的深度網狀檢索，提供帶有精確雙鏈引用的結構化回答。

## 2. 觸發場景
- 用戶輸入 `/query <問題>`
- 用戶詢問："我的筆記裡關於 X 是怎麼說的"、"過去我的決定是什麼"、"查詢 Y 相關的知識"

## 3. 降級策略 (Fallback Mechanism)
若問題屬於純通用知識（例如："什麼是第一性原理"），且檢索 `wiki/index.md` 後確認知識庫內完全沒有相關紀錄，必須明確聲明：
> 本地知識庫中未找到相關內容，以下為通用知識回覆：[直接回答]

---

## 4. 檢索與綜合流水線 (SOP)

### 步驟 1：查閱全局索引
**永遠的第一步**：禁止直接盲目推論。必須先讀取 `wiki/index.md`。
在 index 中依據關鍵字精確定位與問題相關的 Sources、Entities、Concepts 或 Syntheses 頁面連結。

### 步驟 2：深度閱讀目標檔案
選取步驟 1 中篩選出最相關的 1~3 個 Wiki 頁面，使用讀取工具獲取其完整的 Markdown 內容。

### 步驟 3：綜合與回覆 (帶雙鏈引用)
綜合讀取到的本地知識，撰寫深度回答。
- **雙鏈引用規範**：只要引用了某個 Wiki 頁面的內容，必須在文本中自然嵌入 `[[頁面名稱]]` 作為引用標記。
- **避免過度引用**：同一頁面的連結，在同一個段落的首尾各出現一次即可，不需字字連結。
- **精確原文**：若有特定重要決定，使用 Markdown 塊引用 `> 引用內容` 呈現。

### 步驟 4：高價值內容固化 (Synthesis Auto-Trigger)
若產出的回答同時滿足以下條件：
1. 回答長度超過 2 個段落。
2. 內容具有高度跨檔案分析、對比性或總結性（如：產品 naming 策略總結）。
*必須主動詢問用戶*：
> 「這是一個非常有價值的總結，是否需要我將其儲存到 wiki/syntheses/ 目錄？」

用戶點頭同意後，嚴格依照 `WIKI_SCHEMA.md` 的規範在 `wiki/syntheses/` 創立新檔案，並同步在 `wiki/index.md` 進行註冊。

新檔案建立格式範本：
```markdown
---
title: "頁面標題"
type: synthesis
tags: []
sources: []
last_updated: YYYY-MM-DD
---

# 總結內容
```

### 步驟 5：記錄操作日誌
無論最終有沒有生成固化頁面，查詢結束後必須在 `wiki/log.md` 末尾追加一筆紀錄：
```markdown
## [YYYY-MM-DD] query | <查詢問題之簡述>
- **輸出**: <引用頁面列表 或 "即時回答未保存">
```
格式必須完全遵循 `WIKI_SCHEMA.md` 的規範中的範例。

---

## 強制約束
- **禁止憑記憶回答**：必須先檢索知識庫
- **禁止過度引用：同一頁面的訊息在段落首尾引用一次即可
- **禁止靜默回答：知識庫無相關內容時必須聲明
